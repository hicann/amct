# 直转量化

## 作用

这一页只讨论**第一版直转量化实验怎么设计、怎么跑、怎么判读**。

直转量化指：

- 不引入 PTQ 训练
- 直接按目标 bit / dtype 构建量化模型并评测

## 基本原则

1. 默认先做直转量化，再决定是否进入 PTQ
2. 第一版方案优先求简单、可测、可解释，不追求一步到位
3. 先从整网视角定方案，再决定是否缩到 block / unit

## 第一版方案应该回答什么

至少明确：

- 本次量化哪些模块
- 本次暂时不量哪些模块
- 目标 bit / dtype 是什么
- 是否先只做 weight-only
- 是否先跳过 cache 或特别复杂的路径

## 结果记录建议

每次直转量化至少记录：

- `ppl_bf16`
- `ppl_quant`
- `delta = ppl_quant - ppl_bf16`
- 本次量化方案说明

## 第一轮判读

### 结果可接受

满足下面任一条时，可以先认为当前方案可接受：

1. `delta <= 0.2`
2. 当前方案已经满足业务目标

### 结果不可接受

满足下面任一条时，进入下一步粗粒度误差定位：

1. `delta > 0.2`
2. 当前方案虽然可跑，但掉点明显超过预期

### 结果异常

如果出现下面情况，不先当成成功或失败，而是先检查链路：

- 量化后 PPL 明显优于 BF16 很多
- 掉点离谱且和已有经验完全不符
- 同一方案多次复现不稳定

优先检查清单（按 PPL 偏低 / 偏高分两类）见 [metrics-and-thresholds.md](metrics-and-thresholds.md) 的「结果异常时的默认解释」。
