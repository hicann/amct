# 指标与边界

## 当前主指标

当前阶段默认只使用：

- Wikitext PPL

## 当前比较方式

先拿到 BF16 baseline：

- `ppl_bf16`

再比较量化结果：

- `ppl_quant`

当前使用的边界量是：

- `delta = ppl_quant - ppl_bf16`

## 当前默认边界

- 如果 `delta <= 0.2`，认为当前量化方案可接受

这个边界目前只用于：

- 直转量化第一轮判断
- PTQ 结果的粗判

## 当前不使用的规则

当前阶段先不要引入：

- 相对比例阈值
- 多数据集混合边界
- 多指标联合打分

这些规则以后再补。

## 结果异常时的默认解释

### 量化后 PPL 明显比 BF16 更低

优先怀疑：

- 评测链路
- wrapper 实现
- forward / mask 逻辑
- 保存与加载不一致

### 量化后 PPL 明显高得离谱

优先怀疑：

- 量化范围过大
- 某一类模块特别敏感
- 关闭量化后 wrapper 其实不等价
- 量化参数没有正确加载

## 使用原则

1. 先看 BF16 baseline 是否稳定
2. 再看当前方案的 `delta`
3. 不要只报数值，必须给出下一步决策
