# 网络策略

## 作用

这一页只讨论：

- 不同网络结构下，量化方案优先走哪条路径
- 哪些网络类型更适合先做直转
- 哪些网络类型更适合更早引入 PTQ

## Dense 模型

默认建议：

1. 先从整网直转量化开始
2. 如果不达标，再看是 MLP 还是 attention 更敏感
3. 优先在最敏感路径引入 PTQ

## MoE 模型

默认建议：

1. 先看整网方案是否整体可行
2. 如果误差集中在 MoE 路径，再进一步区分：
   - shared 部分
   - routed expert 部分
3. 如果需要，再进入 expert-wise PTQ

## Packed Experts

默认建议：

1. 不要一开始就直接上最细粒度的 expert 级复杂策略
2. 先确认 packed expert 的 unit 拆分、保存和加载路径稳定
3. 再决定是否进入更细粒度 PTQ

## 自定义 Attention

默认建议：

1. 先确认关闭量化后，attention wrapper 与原始浮点实现保持一致
2. 再做直转量化
3. 如果 attention 明显敏感，再优先围绕 attention 做 PTQ 尝试

## 结构级算法

默认建议：

1. 不要把结构级算法当第一选择
2. 只有在直转和普通 PTQ 都无法满足需求，或已知存在结构性 outlier 时，再考虑引入
3. 引入后仍然要先在最小范围验证收益
