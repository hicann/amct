# Skill 调用输入模板

这份模板用于规范用户调用 amct skill 时的输入格式。

使用原则：

- 当用户明确点名某个 skill 或 agent 时，不替用户改成别的 skill。
- 这时先返回该 skill 对应的固定输入模板，让用户补齐信息后再执行。
- 只有当用户没有指定 skill、只是描述目标时，才由 agent 建议最合适的入口。
- 用户输入优先按固定模板给出；如果字段缺失，再由 agent 在最小范围内补默认值或继续确认。

字段标注规则：

- `[必填]`：缺了就无法稳定开始执行。
- `[可选]`：缺了可以先由 agent 结合代码仓、casebook、已有结果或默认规则补判断。
- 除非模板里明确标成 `[必填]`，否则默认都按 `[可选]` 处理。

## 1. 用户点名调用：quant-workflow（量化编排入口）

适用场景：
- 新模型刚接入，希望一整套流程往下走
- 希望先查现状、复用已有结果、再决定下一步

推荐输入格式：

```text
按 quant-workflow，帮我处理 <模型名>。

已知：
- [必填] 权重路径：
- [必填] 期望的 model_name：
- [必填] 远程环境：
- [可选] 模型结构：dense / MoE / 其他
- [可选] 特殊结构：无 / MLA / linear attention / 其他
- [可选] 当前阶段：未知 / 未适配 / 已适配未量化 / 已有量化结果 / 准备 deploy
- [可选] 目标：先打通 BF16 / 完成 model-adapter / 完成第一轮量化 / 一直到 deploy
- [可选] 是否允许复用已有结果：是 / 否
- [可选] 备注：
```

## 9. 当用户没有指定 skill 时的建议规则

- 用户没有指定 skill，只说要走整套流程时，建议 `quant-workflow`。
- 用户没有指定 skill，只说“接入新模型”时，建议 `amct 模型适配`。
- 用户没有指定 skill，只说“做量化实验 / PTQ / 评测量化方案”时，建议 `quant-workflow`（量化实验）。
- 用户没有指定 skill，只说“导出 deploy 权重”时，建议 `Deploy 导出`。
